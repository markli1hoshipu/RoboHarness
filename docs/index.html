RoboHarness

Memory-Driven Orchestration and Chaining of Heterogeneous
Policies for Long-Horizon Robot Planning

Jinbang Huang¹, Yuanzhao Hu¹, Zhiyuan Li¹, Ran Qi¹, Yixin Xiao¹, Zhanguang Zhang¹, Mark Coates², Tongtong Cao¹, Yingxue Zhang¹
¹Huawei Noah's Ark Lab, ²McGill University

Abstract

Long-horizon robotic tasks require instruction following, environmental generalization, logical consistency, and robustness, yet no single policy reliably satisfies all these requirements. Existing policies instead exhibit complementary but bounded capabilities, motivating the joint use of multiple heterogeneous policies. The central challenge is therefore not planning alone, but decomposing tasks and assigning subtasks according to policy capabilities. However, existing long-horizon planning methods are largely built on homogeneous predefined skills with fixed applicability, and therefore lack the ability to reason about uncertain policy boundaries, adaptively route subtasks, or handle distribution mismatch across policy handoffs.

We propose RoboHarness, a unified framework that orchestrates heterogeneous robot policies as reusable agentic skill modules. RoboHarness incorporates auxiliary modules that characterize policy capability boundaries and mediate inter-policy inputs and outputs. It can coordinate independently trained policies — such as VLAs and RL policies — as well as hand-designed systems such as task-and-motion planners, despite differences in assumptions, representations, and execution regimes. Experiments on public benchmarks and real robots show substantial gains in long-horizon planning, zero-shot generalization, and out-of-distribution robustness.

Motivation

Modern robot systems contain many useful policies, but they are rarely designed to work together. A VLA policy may be strong at language-conditioned manipulation but weak under geometric precision constraints. A task-and-motion planner may be precise and logically consistent but brittle when perception or object semantics are uncertain. An RL policy may solve a narrow control subproblem but fail outside its training distribution.

Naively chaining these policies is unreliable because each policy expects states, observations, and action contexts that resemble its own training or design distribution. After one policy completes a subtask, the resulting scene may be valid for the overall task but still incompatible with the next policy. RoboHarness treats this as the central long-horizon planning problem: not only deciding what should happen next, but deciding which policy should do it and how to make the handoff executable.

Key Contributions

🧩

Heterogeneous Policy Orchestration

A unified framework for zero-shot long-horizon planning that encapsulates heterogeneous robot policies as reusable agentic skills and augments them with auxiliary modules for capability characterization and inter-policy mediation — enabling capability-aware task decomposition, adaptive policy routing, and spatially consistent handoff under uncertain capability boundaries.

🌉

The Memory Bridge

A plug-and-play inter-policy chaining technique that combines memory retrieval and spatial distribution learning to preserve stable inter-policy handoffs — without any additional joint retraining of the coordinated policies.

🤖

Empirical Effectiveness

Demonstrated on zero-shot long-horizon tasks by orchestrating heterogeneous policies including VLAs, TAMP systems, and RL policies. Evaluations across multiple benchmarks and real robots show improved long-horizon consistency, out-of-distribution resilience, and zero-shot robustness.

Method Overview

RoboHarness wraps each heterogeneous policy as a callable agentic skill with policy-specific metadata, and uses auxiliary skills to identify capability boundaries and bridge transitions. Given a high-level instruction, it decomposes the task, routes each subtask to a policy operating within its capability region, and constructs a memory-conditioned handoff between successive policies to preserve executability.

Execution loop

During execution, RoboHarness performs six steps:

  • Observe context. Take in the current multimodal context — image observations, instruction text, estimated object poses, robot state, and execution history.
  • Extract signals. Use auxiliary skills to extract task-relevant signals for capability reasoning: visual-semantic similarity, object-pose feasibility, robot-state compatibility, action confidence, and environment quality.
  • Reason about capability boundaries. Compare each candidate policy's memory and capability region against the current task context.
  • Decompose & assign. Decompose the instruction into subtasks according to what the available modules can reliably execute, and assign each to a suitable policy (VLA, task-and-motion planner, RL policy, grasping module, or navigation component).
  • Construct a Memory Bridge. At each policy transition, retrieve in-distribution execution memories and estimate a spatially compatible handoff state for the next policy.
  • Execute & adapt. Execute, monitor, retry, reroute, and update memory when the plan becomes invalid, the environment changes, or a capability mismatch is detected.

Auxiliary Skill Modules

Three categories of auxiliary skills support the orchestration process. They do not replace the robot policies — they provide evidence for routing and handoff decisions.

🔍 Understand Skills

Extract decision-relevant information from multimodal inputs — semantic object information, visual similarity, image quality, scene layout, pose estimates, and robot-state compatibility — as evidence for routing and handoffs.

🧠 Memory Skills

Retrieve and update policy-specific execution memories: demonstrations, behavior-cloning data, rollout trajectories, observations, prompts, end-effector poses, joint states, and execution traces — used to gauge context compatibility and build handoffs.

🌱 Evolution Skills

Adapt the harness online — refine policy metadata, update routing rules, handle unseen objects or task variants, adjust bridge construction, and trigger retry or rerouting when evidence shows the selected policy is no longer appropriate.

The Memory Bridge

The Memory Bridge is the key mechanism for chaining heterogeneous policies without joint retraining; its purpose is to reduce distribution mismatch at the moment execution transfers from one policy to another. For each policy, RoboHarness maintains a memory bank of in-distribution execution data (training data and rollout history). When a handoff is required, the current instruction and observation are encoded by language and visual encoders and compared against the memory bank by cosine similarity. The top retrieved examples reveal which states, object poses, end-effector configurations, and trajectories are compatible with the next policy; RoboHarness then fits a spatial distribution over feasible handoff states and uses it to validate, adjust, or reconstruct the transition into that policy's executable region.

In practice the bridge supports several handoff behaviors:

  • Validate. Confirm the current state is already suitable for the next policy.
  • Adjust. Nudge an object or robot pose before invoking the next policy.
  • Fallback. Select a fallback trajectory or retry from a nearby compatible state.
  • Reroute. Switch to a different policy when bridge confidence is low.

Problem: Policy Capability-Aware Planning (PCAP)

We consider zero-shot long-horizon planning with a library of heterogeneous policies Π = {π₁, …, πN}. Each policy πi is an executable module with its own input space 𝒳i, output space 𝒴i, and capability region 𝒞i ⊆ 𝒪 × 𝒢 — the set of (observation-context, subtask) pairs it can reliably execute. A memory bank 𝓜 = {𝓜₁, …, 𝓜N} stores in-distribution execution data (demonstrations, behavior-cloning data, or rollout trajectories) for each policy.

  • Decompose. Turn instruction I into a subtask sequence τ = (g₁, …, gT).
  • Assign. Choose a policy for each subtask, ρ = (πk₁, …, πkT), with each applied inside its capability region: (ot, gt) ∈ 𝒞kt.
  • Bridge. Build handoff representations z = (z₁, …, zT−1) so one policy's terminal state lands in the next policy's executable region.
  • Objective. Maximize P(oT ⊨ G(I) | o₀, I, Π, 𝓜) subject to capability and memory-conditioned handoff-compatibility constraints.

Plainly, RoboHarness asks three questions at every step:

  1. What subtask should be attempted next?
  2. Which policy is most likely to execute it under the current context?
  3. How should the scene, state, or representation be adjusted so the next policy receives an input it can reliably handle?

This shifts long-horizon planning from fixed skill sequencing to adaptive orchestration under uncertain policy boundaries.

Results

RoboHarness is evaluated along five complementary axes — long-horizon task completion, perturbation robustness, ablation of system components, adaptive policy usage, and real-robot deployment — testing whether it can coordinate heterogeneous policies, preserve handoff consistency, and recover when the scene changes. It beats prior state-of-the-art models without any re-training or fine-tuning.

Ablation study

Removing each RoboHarness component (success rate and progress score, %).

Takeaway: Every component matters. Removing the Memory Bridge drops success 86.0% → 60.4% and removing the Evolution Module → 54.6%; a motion planner alone reaches only 36.4%, and single-policy baselines (Pi05, RL-Open-VLA) fail to complete the tasks (0% success) — confirming that heterogeneous orchestration with memory-bridged handoffs drives the gains.
