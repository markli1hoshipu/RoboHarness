Abstract

Long-horizon robotic tasks require instruction following, environmental generalization, logical consistency, and robustness, yet no single policy reliably satisfies all these requirements. Existing policies instead exhibit complementary but bounded capabilities, motivating the joint use of multiple heterogeneous policies. The central challenge is therefore not planning alone, but decomposing tasks and assigning subtasks according to policy capabilities. However, existing long-horizon planning methods are largely built on homogeneous predefined skills with fixed applicability, and therefore lack the ability to reason about uncertain policy boundaries, adaptively route subtasks, or handle distribution mismatch across policy handoffs.

We propose RoboHarness, a unified framework that orchestrates heterogeneous robot policies as reusable agentic skill modules. RoboHarness incorporates auxiliary modules that characterize policy capability boundaries and mediate inter-policy inputs and outputs. It can coordinate independently trained policies — such as VLAs and RL policies — as well as hand-designed systems such as task-and-motion planners, despite differences in assumptions, representations, and execution regimes. Experiments on public benchmarks and real robots show substantial gains in long-horizon planning, zero-shot generalization, and out-of-distribution robustness.

Key Contributions

🧩

Heterogeneous Policy Orchestration

A unified framework for zero-shot long-horizon planning that encapsulates heterogeneous robot policies as reusable agentic skills and augments them with auxiliary modules for capability characterization and inter-policy mediation — enabling capability-aware task decomposition, adaptive policy routing, and spatially consistent handoff under uncertain capability boundaries.

🌉

The Memory Bridge

A plug-and-play inter-policy chaining technique that combines memory retrieval and spatial distribution learning to preserve stable inter-policy handoffs — without any additional joint retraining of the coordinated policies.

🤖

Empirical Effectiveness

Demonstrated on zero-shot long-horizon tasks by orchestrating heterogeneous policies including VLAs, TAMP systems, and RL policies. Evaluations across multiple benchmarks and real robots show improved long-horizon consistency, out-of-distribution resilience, and zero-shot robustness.

Method Overview

RoboHarness assumes access to existing robot control policies and auxiliary tools, all packaged as callable agentic skills for a coding agent. Given a high-level instruction, it decomposes the task, routes each subtask to a policy operating within its capability region, and bridges the handoff between successive policies to preserve spatial consistency.

Auxiliary Tools

Independently trained or hand-designed policies are augmented by three categories of callable tools:

🔍 Understanding Tools

Extract perceptual, spatial, and semantic information from the environment to support task decomposition and policy selection.

🧠 Memory Tools

Retrieve and update multimodal memories to support decision making and memory-bridge construction across policy handoffs.

🌱 Evolution Tools

Update the harness online by improving harness structure, refining policy metadata, and handling unseen objects, tasks, scenarios, and environments.

Problem: Policy Capability-Aware Planning (PCAP)

We consider zero-shot long-horizon planning with a library of heterogeneous policies Π = {π₁, …, πN}. Each policy πi is an executable module with its own input space 𝒳i, output space 𝒴i, and capability region 𝒞i ⊆ 𝒪 × 𝒢 — the set of (observation-context, subtask) pairs it can reliably execute. A memory bank 𝓜 = {𝓜₁, …, 𝓜N} stores in-distribution execution data (demonstrations, behavior-cloning data, or rollout trajectories) for each policy.

  • Decompose. Turn instruction I into a subtask sequence τ = (g₁, …, gT).
  • Assign. Choose a policy for each subtask, ρ = (πk₁, …, πkT), with each applied inside its capability region: (ot, gt) ∈ 𝒞kt.
  • Bridge. Build handoff representations z = (z₁, …, zT−1) so one policy's terminal state lands in the next policy's executable region.
  • Objective. Maximize P(oT ⊨ G(I) | o₀, I, Π, 𝓜) subject to capability and memory-conditioned handoff-compatibility constraints.

Real Robot Experiments

RoboHarness is deployed on real hardware across four long-horizon manipulation tasks, coordinating heterogeneous policies through capability-aware routing and memory-bridge handoffs.
